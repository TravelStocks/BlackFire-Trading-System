# 连板假设数学建模体系

> 目的：把“开盘竞价幅度 x 对应当日振幅和最低下杀约束”从经验判断，整理成一套可以检验、校准、迭代的研究模型。本文只用于历史样本研究，不构成投资建议。

## 1. 研究对象与边界

研究对象是已经出现连板的股票在连板过程中的日内行为。

核心边界：

- 首板、二板属于“定龙前观察期”，默认记为定龙前例外。
- 正式验证从三板开始，因为定龙通常需要从第三个板以后确认。
- 首板、二板可以保留在表格中作为背景信息，但不进入主规则的满足率统计、OLS 拟合和参数校准。
- 当前网页样本中，三板起验证样本为 49 日，定龙前例外为 18 日。

## 2. 基础变量定义

对每只股票的第 t 个连板交易日，定义：

```text
C_{t-1} = 前一交易日收盘价
O_t     = 当日开盘价
H_t     = 当日最高价
L_t     = 当日最低价
Close_t = 当日收盘价
Vol_t   = 当日成交量，单位：手
T_t     = 当日换手率，单位：%
stage_t = 当前连板阶段，首板=1，二板=2，三板=3 ...
```

核心衍生变量：

```text
x_t = O_t / C_{t-1} - 1
A_t = (H_t - L_t) / C_{t-1}
M_t = L_t / C_{t-1} - 1
U_t = x_t - M_t
V_t = Vol_t / 10000
```

含义：

- `x_t`：开盘竞价幅度。
- `A_t`：当日振幅。
- `M_t`：最低下杀位置，按前收为基准。
- `U_t`：从开盘到最低点的下杀幅度。
- `V_t`：当日成交量，单位为万手。

## 3. 先验约束包络

Excel 表给出的是人工先验，不是最终真理。

把表格转换成两条分段线性函数：

```text
F0(x_t) = 给定开盘竞价幅度 x_t 时，允许的当日振幅上限
G0(x_t) = 给定开盘竞价幅度 x_t 时，允许的最低下杀线
```

主假设：

```text
H0: A_t <= F0(x_t) 且 M_t >= G0(x_t)
```

解释：

- 如果开盘越强，理论上日内波动空间应越窄。
- 如果开盘较弱，允许更大的日内分歧，但最低下杀仍应受约束。
- 超出约束并不必然代表假设失效，可能是爆量、换手、弱转强或大分歧日。

## 4. 定龙前例外规则

首板、二板不进入核心模型：

```text
if stage_t <= 2:
    result_t = 定龙前例外
    exclude_from_calibration = true
```

原因：

- 首板通常是题材发现或消息刺激阶段。
- 二板通常仍处于资金确认和市场试错阶段。
- 从三板开始，股票更可能进入“龙头确认/定龙验证”区间。
- 如果把首二板纳入校准，容易把早期随机波动误当作龙头规律。

## 5. 残差与破坏度

为了严肃判断，不只用“满足/不满足”二分，而要度量破坏来自哪里。

振幅残差：

```text
rA_t = A_t - F0(x_t)
```

最低下杀残差：

```text
rL_t = G0(x_t) - M_t
```

总破坏度：

```text
B_t = max(0, rA_t) + max(0, rL_t)
```

解释：

- `rA_t <= 0` 表示当日振幅没有超过阈值。
- `rL_t <= 0` 表示最低下杀没有破线。
- `B_t = 0` 表示完全满足约束。
- `B_t > 0` 表示至少有一个约束被破坏。

## 6. 三板起判定体系

对三板及以后样本：

```text
if B_t = 0:
    result_t = 满足
elif B_t 接近 0:
    result_t = 临界
elif B_t > 0 且 E_t 高:
    result_t = 解释性例外
else:
    result_t = 规则失效样本
```

当前网页中的近似口径：

- 满足：振幅和最低下杀都未破坏约束。
- 临界：满足但距离阈值很近。
- 解释性例外：破坏约束，但当日成交量、换手或波动显示明显分歧。
- 不满足：破坏约束，且没有足够分歧解释。

## 7. 量能分歧例外门

量能分歧只看当日成交量，不使用成交量比例。

定义例外压力：

```text
E_t = sigmoid(
    -1.4
    + 0.22 * ln(1 + V_t)
    + 0.055 * max(0, T_t - 10)
    + 0.18 * max(0, rA_t)
    + 0.20 * max(0, rL_t)
)
```

其中：

```text
sigmoid(z) = 1 / (1 + exp(-z))
```

解释：

- `V_t` 越大，当日分歧越可能是真实换手。
- `T_t` 超过 10% 后，换手越高，分歧解释力越强。
- `rA_t` 越大，说明振幅破坏越明显。
- `rL_t` 越大，说明最低下杀破线越明显。
- `E_t` 越高，越倾向把样本归为解释性例外，而不是直接判定规则失效。

注意：这里的系数是初始研究参数，不是最终统计真值。后续应通过更多样本校准。

## 8. 参数校准方法

初始阶段使用 Excel 表作为 `F0/G0` 先验。

样本增加后，应升级为状态条件函数：

```text
F(x | state) = 给定状态下的振幅上限分位函数
G(x | state) = 给定状态下的最低下杀分位函数
```

状态变量可以包括：

- 连板阶段：三板、四到六板、七板以上。
- 题材强度：主线、支线、补涨、轮动。
- 市场情绪：上升、高潮、分歧、退潮。
- 指数环境：强势、震荡、弱势。
- 当日成交量、换手率。

推荐校准：

```text
F(x | state) = Quantile(A_t | x_t≈x, state, q=0.80~0.90)
G(x | state) = Quantile(M_t | x_t≈x, state, q=0.10~0.20)
```

含义：

- `F` 用高分位，估计多数正常样本不会超过的振幅上沿。
- `G` 用低分位，估计多数正常样本不会跌破的最低下沿。

## 9. OLS 探索模型

OLS 不作为最终交易规则，只用于观察方向。

振幅探索模型：

```text
A_t = β0 + β1*x_t + β2*stage_t + β3*ln(1+V_t) + β4*T_t + ε_t
```

最低点探索模型：

```text
M_t = γ0 + γ1*x_t + γ2*stage_t + γ3*ln(1+V_t) + γ4*T_t + η_t
```

用途：

- 观察开盘强度与振幅的关系。
- 观察连板高度是否收敛或放大波动。
- 观察当日成交量和换手是否解释分歧。
- 发现样本中可能被遗漏的状态变量。

## 10. 验证协议

严肃验证要避免“用同一批样本调参并宣布有效”。

建议：

1. 时间滚动验证  
   用前一段样本训练，用后一段样本检验。

2. 跨题材验证  
   电力、医药、电池、储能等题材分组评估。

3. 跨阶段验证  
   三板、四到六板、七板以上分别计算满足率和例外率。

4. 市场状态分层  
   情绪上升期、高潮期、分歧期、退潮期分开检验。

5. 分钟线复核  
   当前 `M_t` 使用日低价代理。严谨版本应接入分钟线，验证真实盘中路径下杀。

## 11. 输出分级

最终每个三板起样本可以输出五类：

```text
定龙前例外：stage <= 2
满足：B_t = 0
临界：B_t ≈ 0，接近阈值
解释性例外：B_t > 0 且 E_t 高
规则失效：B_t > 0 且 E_t 低
```

对股票连板段可以输出：

- 三板起满足率。
- 解释性例外日数量。
- 最大破坏日。
- 平均破坏度。
- 当前选中日例外压力。

## 12. 当前体系的局限

当前版本仍有几个限制：

- 样本量小，只有五只股票的连板段。
- 分时数据尚未接入，最低下杀用日低价代理。
- 当日成交量是绝对量，不同行业和流通盘之间仍可能需要流通市值归一化。
- 市场情绪、题材强度尚未量化进入模型。
- Excel 阈值表仍是人工先验，需要更多样本校准。

## 13. 一句话总结

这套体系的核心是：

```text
首二板不算定龙；三板起用开盘强度决定日内振幅和下杀包络；
破坏包络时，再用当日成交量、换手和破坏程度判断是否属于解释性分歧；
随着样本增加，用分层分位数和滚动验证持续校准包络函数。
```

## 14. 冲高回落止盈模型

前面的模型主要解决止损问题：开盘后向下不能破哪里。

止盈模型解决另一个问题：开盘后先向上冲，冲到某个点后第一次回落，回落多少以内仍然算健康，超过多少说明承接失败。

定义：

```text
H1_t = 开盘后第一次有效冲高点
P1_t = H1_t 后第一次有效回落点
h_t  = H1_t / C_{t-1} - 1
p_t  = P1_t / C_{t-1} - 1
I_t  = h_t - x_t
D_t  = h_t - p_t
R_t  = D_t / I_t
```

含义：

- `I_t`：开盘到第一次冲高的上冲幅度。
- `D_t`：第一次冲高后吐回的幅度。
- `R_t`：回吐比例，也就是上冲收益被吐回了多少。

## 15. 动态安全回落线

开盘越强，第一次回落应该越浅；开盘越弱，允许回吐空间可以略大。

定义安全回吐比例：

```text
alpha_safe(x_t) = clamp(0.45 - 0.015 * max(x_t, 0), 0.28, 0.45)
```

定义硬性回吐比例：

```text
alpha_hard(x_t) = clamp(0.62 - 0.012 * max(x_t, 0), 0.45, 0.62)
```

安全回落线：

```text
S_t = h_t - alpha_safe(x_t) * I_t
```

不能接受线：

```text
b_t = clamp(0.15 * I_t, 0.30, 1.20)
K_t = max(
    h_t - alpha_hard(x_t) * I_t,
    x_t - b_t,
    G0(x_t)
)
```

解释：

- `S_t` 是健康回落线。第一次回落不破它，说明冲高后的承接仍然强。
- `K_t` 是硬保护线。第一次回落跌破它，说明冲高收益被过度吐回。
- `x_t - b_t` 是开盘守门线，防止强开后直接跌回开盘下方太多。
- `G0(x_t)` 是原止损模型里的最低下杀线，防止止盈模型放得比止损还宽。

## 16. 止盈判定

对三板及以后的龙头样本：

```text
if p_t >= S_t:
    result = 强承接
elif K_t <= p_t < S_t:
    result = 偏弱观察
else:
    result = 不能接受
```

操作含义：

- 强承接：第一次回落仍在安全回落线之上，可以继续看二次上攻。
- 偏弱观察：第一次回落超过安全线，但没有跌破硬保护线，适合降低仓位或收紧止盈。
- 不能接受：第一次回落跌破硬保护线，说明冲高失败，应按模型触发止盈/保护。

## 17. 与止损模型的关系

止损模型：

```text
开盘后没有有效上冲时，关注 G0(x_t) 和 F0(x_t)
```

止盈模型：

```text
开盘后已经有效上冲时，关注 S_t 和 K_t
```

两者合并后：

```text
开盘前设 G0(x_t) 作为防守底线；
开盘后若先冲高，则把保护线动态上移到 K_t；
若第一次回落仍在 S_t 之上，则保留二次上攻预期；
若跌破 K_t，则优先保护利润。
```
